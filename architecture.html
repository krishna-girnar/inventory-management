Data Flow — Filter Change Example
1

User selects a filter

FilterBar.vue dropdown emits change. useFilters() singleton updates selectedLocation.value = 'Tokyo'.

2

View watcher fires

Each view watches the relevant filter refs. watch([selectedLocation, selectedCategory], loadData) triggers loadData().

3

API call constructed

api.js builds URLSearchParams, skipping 'all' values. Sends GET /api/orders?warehouse=Tokyo.

4

FastAPI filters in-memory data

apply_filters(orders, warehouse='Tokyo') iterates the loaded Python list. For months/quarters, filter_by_month() parses dates.

5

Pydantic validates & serialises

Response is coerced through List[Order] Pydantic model before returning JSON to the client.

6

Vue reacts

Response assigned to orders.value. Computed properties (filteredItems, status counts) recalculate. DOM updates reactively.

Key Architectural Decisions

In-memory data, no database

All JSON is loaded into Python dicts at startup. Removes infrastructure complexity for a demo, but changes don't persist across restarts.

Server-side filtering

All filter logic lives in apply_filters() on the backend. The frontend simply passes query params — clean separation of concerns.

Singleton composable for filters

useFilters() returns module-level refs shared across every view — a lightweight alternative to Pinia/Vuex for cross-component state.

Computed properties over mutations

Raw data stays in refs (allOrders). Derived values (counts, totals, colours) are computed() — no manual re-syncing needed.

Centralised API client

A single api.js owns all Axios calls and filter-to-params conversion. Views call named methods, not raw URLs.

Custom SVG charts

No charting library dependency. Charts are hand-written SVG / CSS Grid — simpler bundle, full styling control.

CORS allow-all

FastAPI CORS middleware allows all origins for dev convenience. Would be locked down per-origin in production.

i18n via composable

useI18n() loads locale JSON and exposes a t('key') helper. English and Japanese supported with currency/number formatting.
